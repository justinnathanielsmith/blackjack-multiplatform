# JetBrains Amper Project Manager

This skill provides the procedural knowledge needed to correctly manage JetBrains Amper projects. Amper uses a convention-over-configuration, declarative YAML-based approach instead of Gradle build scripts.

## Core Concepts & File Structure

- `project.yaml`: The root configuration file that defines all modules within the project.
- `module.yaml`: The configuration file inside each module defining its product type, settings, and dependencies.
- To execute a build, run `./amper build` at the project root.
- To use an Amper project, you should stick to standard layout constraints. 
  - Android manifests go in `<module-root>/src/AndroidManifest.xml` (not `src/main/`).
  - Android resources go in `<module-root>/res/` (not `src/main/res/`).
  - Native iOS frameworks generated by Amper for Swift consumption are imported using `import KotlinModules`. Do not attempt to override `native.frameworkName` in KMP library modules.

## Dependency Management & Version Catalogs

Amper relies heavily on string prefixes within the `module.yaml` file to resolve dependencies.

### 1. Version Catalogs (libs.versions.toml)

Amper automatically supports Gradle version catalogs located at `<project-root>/gradle/libs.versions.toml`.

- **To reference a library:** Use the `$libs.` prefix followed by the library alias from the catalog (with dots instead of dashes).
  - *Example:* If your TOML has `kotlinx-coroutines-core = { ... }`, reference it as `$libs.kotlinx.coroutines.core` in `module.yaml`.
- **Bundles:** Amper **DOES NOT** currently support referencing TOML `[bundles]` directly (e.g., `$libs.bundles.room` will fail). You must specify each library individually.
- **Exporting (Transitive dependencies):** To make a dependency available to consumers of the module, append `: exported` to the end of the string.
  - *Example:* `- ../shared/core: exported`

### 2. Compose Multiplatform

When using Compose Multiplatform, you must explicitly enable it and use the built-in `$compose` catalog. Do not use external maven coordinates for JetBrains Compose artifacts if you can avoid it.

**Enabling Compose:**
It must be enabled under the `settings` block, properly nested:
```yaml
settings:
  compose:
    enabled: true
```

**Referencing Compose Libraries:**
Use the built-in `$compose.` prefix.
```yaml
dependencies:
  - $compose.material3
  - $compose.ui
  - $compose.resources
  - $compose.desktop.currentOs  # Desktop only
```

## Example `module.yaml` (Shared UI Library)

```yaml
product:
  type: lib
  platforms: [jvm, android, iosArm64, iosSimulatorArm64, iosX64]

settings:
  compose:
    enabled: true

dependencies:
  - ../shared/core: exported
  - $libs.decompose
  - $libs.coil
  - $compose.material3
  - $compose.resources
```

## Example `module.yaml` (Android App)

```yaml
product: android/app

settings:
  compose:
    enabled: true
  android:
    applicationId: com.example.app
    namespace: com.example.app
    compileSdk: 34
    minSdk: 26
    targetSdk: 34

dependencies:
  - ../sharedUI
  - $libs.androidx.activity.compose
```
